  • Instalación y configuración básica

    La configuración básica de ediarum hace disponible un entorno de trabajo para ediciones filológicas. Después de una instalación y configuración existosa, los editores y editoras pueden crear datos XML conforme a TEI con registros, notas etc. en un entorno visual que es similar al de Word.

    • Sobre esta guía

      La configuración básica primero hace disponible ediarum en la versión estándar, sin ninguna configuración específica de su proyecto editorial y sin que necesite conocimientos de programación.

    • Resumen de la configuración básica

      ediarum está disponible en GitHub, en forma de componentes de configuración para Oxygen XML Author (los llamados frameworks) y para la base de datos XML eXist-db (las llamadas aplicaciones).

    • Tips para la instalación de eXist-db y Oxygen

      Las guías de instalación para eXist-db y Oxygen se encuentran en sus sitios web correspondientes. Los consejos adicionales que se han recopilado aquí se basan en la experiencia de los desarrolladores y desarrolladoras de ediarum y pueden ser útiles para los problemas típicos de instalación.

    • Instalar y configurar la base de datos

      Instala la aplicación eXist ediarum.DB. Usando esta aplicación, crea su propio proyecto en su base de datos que contiene todos las configuraciones necesarias para la interacción entre la base de datos y Oxygen XML Author.

    • Configurar el entorno de trabajo en Oxygen XML Author

      Conecta Oxygen XML Author con eXist-db (la base de datos) y hace disponible todas las funcionalidades contenidas en ediarum.

      • Conectar Oxygen a la base de datos

        Configura la conexión para el acceso de lectura y escritura desde la interfaz de entrada a los archivos XML. La conexión WebDAV evita conflictos de escritura entre diferentes usuarios y usuarias.

      • Instalar frameworks de ediarum

        Se integran dos frameworks específicos de ediarum en Oxygen para que las funcionalidades de ediarum estén disponibles en su entorno editorial de Oxygen.

      • Crear un proyecto en el Oxygen XML Author

        Para la búsqueda en los registros se necesita una conexión a la base de datos separada. Se guarda en un proyecto de Oxygen utilizando las llamadas “Editor Variables” (variables de editor).

  • Configuración específica de ediciones

    Si su edicion requiere una definición de esquema XML más completo que el que cubre ediarum, puede extender el documento esquema y el entorno de trabajo. Además, puede crear registros adicionales.

Conectar Oxygen a la base de datos

Configura la conexión para el acceso de lectura y escritura desde la interfaz de entrada a los archivos XML. La conexión WebDAV evita conflictos de escritura entre diferentes usuarios y usuarias.

  1. Abra el Oxygen XML Author.
  2. Seleccione el menú Options > Preferences
  3. En la ventana de Preferences que se ha abierto, seleccione la entrada Data Sources en la columna de la izquierda.
  4. Haga clic en el botón “+” en la mitad inferior (Connections) para crear una nueva conexión.
  5. Rellene los campos en la ventana Connection que se ha abierto.


    • Name: Nombre de esta conexión, por ejemplo, miEdición. Sirve para distinguir esta conexión de otras conexiones a la base de datos que ha definido en Oxygen. No tiene que ser idéntico con el nombre del proyecto en la eXist-db.
    • Data Source: WebDAV (S)FTP
    • WebDAV: Ruta a los datos de su proyecto en su instancia eXist, siguiendo el patrón:: http://HOST:PORT/webdav/db/projects/PROYECTO/data. Por ejemplo:
      • https://miInstituto.de/exist/webdav/db/projects/miEdicion/data
      • http://localhost:8080/exist/webdav/db/projects/miEdicion/data
    • User, Password: Detalles de la cuenta de usuario que ha creado en la aplicación ediarum.DB en eXist-db (Verwaltung > Projekte > Projektkonfiguration > Benutzer, “Gestión → Proyectos → Configuración de proyectos → Usuarios”).
  6. Haga clic en OK.
  7. Compruebe la conexión a la base de datos:
    1. Seleccione el menú Window > Show View > Data Source Explorer.
      La conexión establecida se muestra en el explorador de fuentes de datos (aquí: miEdición).
    2. Haga clic en la flecha al lado de la conexión para abrir el arból de carpetas.
      La estructura es idéntica al arból de carpetas que se muestra en eXist-db en la página principal de su proyecto (en la aplicación ediarum.DB: Verwaltung > Projekte, “Gestión → Proyectos”). Tiene acceso a las carpetas Briefe (“Cartas”) y Register (“Registros”) y puede cambiar el nombre de las mismas.

Si las carpetas no se pueden expandir y aparece un mensaje de error:

  1. Compruebe la exactitud de sus datos en la ventana Connection, en particular:
    • WebDAV: ¿Ha introducido la ruta hasta el nivel .../data?
    • User, Password: ¿Son los detalles idénticos a los de la aplicación ediarum.DB in eXist-db?
    • Disponibilidad: También puede comprobar la exactitud y disponibilidad abriendo la ruta correspondiente en su navegador (introduciendo el usuario y la contraseña). Su navegador debería entonces mostrarle una salida XML con los detalles de la carpeta.
  2. Haga clic con el botón derecho del ratón en su edición en el Data Source Explorer y haga clic en Refresh en el menú de contexto.
