  • Instalación y configuración básica

    La configuración básica de ediarum hace disponible un entorno de trabajo para ediciones filológicas. Después de una instalación y configuración existosa, los editores y editoras pueden crear datos XML conforme a TEI con registros, notas etc. en un entorno visual que es similar al de Word.

    • Sobre esta guía

      La configuración básica primero hace disponible ediarum en la versión estándar, sin ninguna configuración específica de su proyecto editorial y sin que necesite conocimientos de programación.

    • Resumen de la configuración básica

      ediarum está disponible en GitHub, en forma de componentes de configuración para Oxygen XML Author (los llamados frameworks) y para la base de datos XML eXist-db (las llamadas aplicaciones).

    • Tips para la instalación de eXist-db y Oxygen

      Las guías de instalación para eXist-db y Oxygen se encuentran en sus sitios web correspondientes. Los consejos adicionales que se han recopilado aquí se basan en la experiencia de los desarrolladores y desarrolladoras de ediarum y pueden ser útiles para los problemas típicos de instalación.

    • Instalar y configurar la base de datos

      Instala la aplicación eXist ediarum.DB. Usando esta aplicación, crea su propio proyecto en su base de datos que contiene todos las configuraciones necesarias para la interacción entre la base de datos y Oxygen XML Author.

    • Configurar el entorno de trabajo en Oxygen XML Author

      Conecta Oxygen XML Author con eXist-db (la base de datos) y hace disponible todas las funcionalidades contenidas en ediarum.

      • Conectar Oxygen a la base de datos

        Configura la conexión para el acceso de lectura y escritura desde la interfaz de entrada a los archivos XML. La conexión WebDAV evita conflictos de escritura entre diferentes usuarios y usuarias.

      • Instalar frameworks de ediarum

        Se integran dos frameworks específicos de ediarum en Oxygen para que las funcionalidades de ediarum estén disponibles en su entorno editorial de Oxygen.

      • Crear un proyecto en el Oxygen XML Author

        Para la búsqueda en los registros se necesita una conexión a la base de datos separada. Se guarda en un proyecto de Oxygen utilizando las llamadas “Editor Variables” (variables de editor).

  • Configuración específica de ediciones

    Si su edicion requiere una definición de esquema XML más completo que el que cubre ediarum, puede extender el documento esquema y el entorno de trabajo. Además, puede crear registros adicionales.

Instalar frameworks de ediarum

Se integran dos frameworks específicos de ediarum en Oxygen para que las funcionalidades de ediarum estén disponibles en su entorno editorial de Oxygen.

Resumen del procedimiento:

  • Descarga los dos frameworks de ediarum a su ordenador local.
  • Cambia la carpeta predeterminada en la que Oxygen XML Author gestiona los frameworks. De esta manera se evitan problemas con los derechos de acceso por parte del sistema operativo (especialmente Windows).
  • Crea un framework nuevo para su edición. Crea este framework como una extensión de ediarum.BASE.edit para que tenga acceso a todas las funcionalidades de ediarum. Después de terminar la configuración básica, puede guardar las configuraciones específicas de su entorno editorial dentro de este framework.

Conocimientos necesarios:

  • Conocimientos básicos de GitHub
  1. Cree una carpeta para todos los frameworks de Oxygen en su sistema de archivos local .
  2. Descarga a la nueva carpeta de frameworks los siguientes repositorios de GitHub (en GitHub: pulse el botón Latest release, guarda el archivo ZIP en una ubicación local y extraigalo):
    1. https://github.com/ediarum/ediarum.BASE.edit/releases
    2. https://github.com/ediarum/ediarum.REGISTER.edit/releases
  3. Cree en la carpeta de frameworks un directorio adicional, donde puede agregar luego el framework específico de su edición.
    Recomendación: Mantenga ediarum como la primera parte del nombre de la carpeta para indicar que el framework específico de su edición es una extensión de ediarum. Por ejemplo: ediarum.miEdición.edit
  4. Asegúrese de que la carpeta de frameworks tenga la siguiente estructura:
    • frameworks
      • ediarum.BASE.edit
        • css
        • fonts
        • icons
        • resources
        • schemata
        • templates
      • ediarum.REGISTER.edit
        • css
        • resources
        • schema
      • ediarum.miEdición.edit
  5. Cambie la ubicación de almacenamiento de frameworks en Oxygen XML Author, de modo que la ubicación nueva apunte a la carpeta que contiene todas las carpetas de framework.
    1. Seleccione el menú Options > Preferences en Oxygen.
    2. En la ventana de Preferences abierta, seleccione la página Document Type Association > Locations.
    3. Active la opción Custom.
    4. Haga clic en el icono de carpeta para seleccionar la ruta.
  6. Haga clic en OK para cerrar la ventana Preferences.
  7. Reinicie Oxygen XML Author.

    En la página Options > Preferences > Document Type Association se muestran ambos frameworks de ediarum.

  8. Cree un framework extendido para su edición.
    1. Seleccione el framework ediarum.BASE.edit en la página Document Type Association (¡No haga doble clic!) y haga clic en Extend.
    2. Introduzca el nombre del framework de su edición en el campo Name:.
      Recomendación: Utilice el mismo nombre que ha usado para la carpeta del framework de su edición en el sistema de archivos. Por ejemplo: ediarum.miEdición.edit
    3. Active la opción External para Storage: .
    4. Indique la ruta hasta el archivo .framework que se encuentra dentro de la carpeta de su edición. Por ejemplo: ediarum.miEdición.edit/ediarum.miedicion.edit.framework. Utilice el icono de carpeta para seleccionar el archivo.
  9. Haga clic en OK para cerrar la ventana de Preferences.
